Downloading QTLtools

Hereafter, the different versions of QTLtools available for download:

Version Description
Last development version (not recommended) This version is a tar ball of the last version of the code. We recommend instead to use the last release version below. Alternatively to this tar ball, you can also get the lastest version of the source code with by running git clone https://github.com/qtltools/qtltools.git master .
v1.XXX (Binary + source code) This is the last stable version of the code with a static linux binary that should run on most linux plateforms and a binary for MacOSX >= 10.6.

Compiling QTLtools

If the provided binaries does not directly work, QTLtools needs to be re-compiled on your plateform. For this, you may need to install few standard libraries required by QTLtools if you do not have them already installed.


The GNU Scientific Library

This library is usually installed by default on most linux plateforms, otherwise instructions to install it can be found here.


The Boost C++ library

This collection of C++ libraries is also usually installed by default on most computers. Detailled instructions to install it can be found here.


The standalone Rmath library

This library provided by R contains all basic functions to simulate from standard probability distributions (ex: qnorm, pnorm, rnorm, etc ...). Instructions to install it can be found here. A pre-compiled version can be directly obtained by installing the package r-mathlib on Debian/Ubuntu or libRmath-dev on Fedora/RedHat. Otherwise, proceed as follows:

First download the R source code from here.

Then, move the tgz in the folder where you want to install the library (e.g. $(HOME)/Libraries), and run the following commands:

tar xzvf R-3.2.2.tar.gz
cd R-3.2.2/
./configure
cd src/nmath/standalone/
make

This generates a library file libRmath.a that you will need as desribed below.


The standalone Rmath library

This library written in C allows reading and parsing efficiently VCF, BCF, BAM and BED file. This is used by tools such as samtools or bcftools for example. Download the library from here. Then, move the tgz in the folder you want to install the library and run the following commands:

tar xjvf htslib-1.3.1.tar.bz2
cd htslib-1.3.1/
./configure
make

This generates a library file libhts.a that you will need as desribed below.


Modify the QTLtools makefile

Once all libraries installed, just go in the FastQTL folder and compile the code using:

make

This creates a binary of the program in the bin folder nammed fastQTL.
